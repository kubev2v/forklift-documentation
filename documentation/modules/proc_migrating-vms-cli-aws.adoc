= Running an {aws} {ec2} migration from the command-line

[role="_abstract"]
You can migrate from an {aws} {ec2} source provider by using the command-line interface (CLI).

.Prerequisites

* {aws} {ec2} source provider configured
* {virt} destination provider configured
* {ec2} instances must be stopped before migration

.Procedure

. Create the {aws} {ec2} source provider:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv create provider <provider_name> \
     --type ec2 \
     --ec2-region <region> \
     --username $EC2_KEY \
     --password $EC2_SECRET \
     --provider-insecure-skip-tls
----
+
The `--provider-insecure-skip-tls` flag is optional. Add this flag to skip TLS certificate validation.
+
where:
+
`<provider_name>`::
Specifies the name of the {aws} {ec2} provider.
+
`<region>`::
Specifies the {aws} region where your {ec2} instances are located, for example, `us-east-1` or `us-east-2`. This must be the same region as your {rosa} or OSD cluster.
+
`$EC2_KEY`::
Specifies the {aws} access key ID. Set this as an environment variable to avoid exposing credentials in command history.
+
`$EC2_SECRET`::
Specifies the {aws} secret access key. Set this as an environment variable to avoid exposing credentials in command history.

. Create the {virt} destination provider:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv create provider host --type openshift
----
+
This creates a provider named `host` for the local OpenShift cluster.

. Verify the providers are ready:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv get providers
----
+
Both providers should show `STATUS` as `Ready`.

. Get the inventory of {ec2} instances:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv get inventory vms <provider_name>
----
+
This lists all {ec2} instances discovered by the provider, including their state and platform information.

. Create network and storage maps as needed. For more information, see link:{mtv-plan}assembly_planning-migration-aws_mtv#proc_creating-form-based-network-maps-ui-aws_aws[Creating ownerless network maps from the form page in the {project-short} UI] and link:{mtv-plan}assembly_planning-migration-aws_mtv#proc_creating-form-based-storage-maps-ui-aws_aws[Creating ownerless storage maps from the form page in the {project-short} UI].
+

. Create a migration plan:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv create plan <plan_name> \
     --source <provider_name> \
     --vms <instance_id_1>,<instance_id_2>
----
+
where:
+
`<plan_name>`::
Specifies the name of the migration plan.
+
`<provider_name>`::
Specifies the name of the {aws} {ec2} source provider.
+
`<instance_id_1>,<instance_id_2>`::
Specifies one or more {ec2} instance IDs to migrate. Separate multiple instance IDs with commas.
+
[NOTE]
====
If you specify `--warm` when creating a plan, {project-short} automatically changes it to cold migration because {ec2} only supports cold migration.
====

. Start the migration:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv start plan <plan_name>
----

. Monitor the migration progress:
+
[source,terminal,subs="attributes+"]
----
$ {oc} mtv get plans
----
+
This shows the migration status and progress for all plans.

. View detailed migration information:
+
[source,terminal,subs="attributes+"]
----
$ {oc} get vmimports
----
+
This shows the detailed phases of the VM import process.
