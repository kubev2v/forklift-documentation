= Error messages

[role="_abstract"]
Review common error messages encountered during migration. For detailed resolution steps, see the linked troubleshooting procedures.

[cols="2,3,1",options="header"]
.Common migration error messages
|===
|Error message |Description |Resolution

|`Warm import retry limit reached`
|Displayed during a warm migration when a {vmw} virtual machine (VM) has reached the maximum number (32) of changed block tracking (CBT) snapshots during the precopy stage.
|xref:proc_troubleshooting-warm-import-retry-limit_{context}[Resolving warm import retry limit errors]

|`Unable to resize disk image to required size`
|Displayed when migration fails because a virtual machine on the target provider uses persistent volumes with an EXT4 file system on block storage. CDI assumes a default amount of space consumed by the file system, but this does not include the reserved space for the root partition.
|xref:proc_troubleshooting-resize-disk-image_{context}[Resolving disk resize errors]

|`An error occurred while taking a snapshot: Failed to restart the virtual machine`
|Displayed during warm migration of Microsoft Windows VMs when the Volume Shadow Copy Service (VSS) is not running on the guest operating system.
|link:{mtv-plan}assembly_provider-specific-requirements-for-migration_mtv#ref_vmware-prerequisites_mtv[{vmw} prerequisites]

|`ConnectionTestFailed`
|Displayed when creating an Open Virtual Appliance (OVA) provider. Error messages might be displayed before the provider status changes to `Ready``.
|xref:proc_troubleshooting-connection-test-failed-ova_{context}[Resolving OVA connection test errors]

|`Error pulling VDDK image`
|Displayed when a provider in a namespace different from +{namespace}+ cannot access the VDDK image.
|xref:proc_troubleshooting-vddk-image-pull-error_{context}[Resolving VDDK image pull errors]

|`Manual SSH key configuration required`, `Failed to connect via SSH`, or `SSH connection timeout`
|Displayed when {project-short} cannot establish an SSH connection to the ESXi host during storage copy offload operations.
|xref:ref_troubleshooting-storage-copy-offload_{context}[Troubleshooting storage copy offload]

|`VM name validation failed: Invalid DNS name`
|Displayed when a source VM name does not comply with Kubernetes DNS naming requirements.
|link:{mtv-plan}assembly_migrating-vms-web-console_mtv#proc_renaming-vms-for-migration_mtv[Renaming virtual machines]

|`Cannot derive SVM to use; please specify SVM in config file`
|Displayed during storage copy offload operations with NetApp storage when ONTAP is not configured correctly.
|xref:ref_troubleshooting-storage-copy-offload_{context}[Troubleshooting storage copy offload]

|`Migration failed: VDDK image required for vSAN storage`
|Displayed when a VM is backed by {vmw} vSAN storage and no VDDK image is configured. VDDK is mandatory for vSAN migrations.
|xref:proc_troubleshooting-vddk-required-vsan_{context}[Resolving VDDK vSAN errors]
|===
