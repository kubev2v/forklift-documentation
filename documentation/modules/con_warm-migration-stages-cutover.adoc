= Cutover stage

[role="_abstract"]
The VMs are shut down during the cutover stage and the remaining data is migrated. Data stored in RAM is not migrated.

You can start the cutover stage manually by using the {project-short} console or you can schedule a cutover time in the `Migration` custom resource (CR).

The duration of the cutover stage depends on how much data has changed since the last snapshot was created during the precopy stage.
