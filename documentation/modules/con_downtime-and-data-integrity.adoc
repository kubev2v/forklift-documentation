= Minimizing downtime while preserving data integrity

[role="_abstract"]
{project-first} supports features that reduce migration cutover downtime while applying filesystem and error-handling integrity checks where the migration path supports them.

[id="warm-migration-final-downtime_{context}"]
== Warm migration

{project-short} supports warm migrations, which use snapshots to migrate underlying disks while the source virtual machine (VM) remains operational. This approach significantly reduces final downtime.

[id="sco-and-downtime_{context}"]
== Storage copy offload

The storage copy offload feature transfers data directly within the storage array, reducing migration time.
