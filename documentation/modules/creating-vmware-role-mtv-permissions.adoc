= Creating a VMware role to apply MTV permissions

You can create a role in VMware to apply permissions for {project-first}, using one of the following two approaches:

* At the user or group level
* At the object level

include::snip_vm-role-user-group.adoc[]
include::snip_vm-role-user-object.adoc[]

For detailed instructions, see VMware documentation.



* At the user or group level: In the data center, use the role to apply permissions to relevant users or groups, using the *Propagate to children* option.

* At the object level: Use the role to grant permissions to each relevant vSphere logical object, for example, host, vSphere cluster, data center, or network, involved in migration
