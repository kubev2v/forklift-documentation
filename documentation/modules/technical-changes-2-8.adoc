= Technical changes

{project-first} 2.8 has the following technical changes:

.Shortened plan durations with scheduler optimizations in warm migrations

When you start several warm migration plans concurrently, the scheduler is optimised to schedule the remaining migration plans while completing the migration plan that contains a large disk transfer. link:https://issues.redhat.com/browse/MTV-1574[(MTV-1574)]
