= Understanding Deep Inspection validation results

[role="_abstract"]
You can use Deep Inspection validation results to assess migration feasibility and prepare virtual machines for conversion. Review the concerns organized by category and severity to address critical issues before migration.

== Validation concerns

Deep Inspection can identify concerns that are organized by category and severity:

Changed Block Tracking (CBT) not enabled::
CBT is not enabled on the source VM. This does not prevent migration. However, it can impact performance during warm migrations.

Unsupported operating system::
The guest operating system is not supported for migration to {virt}. Check the {project-short} compatibility matrix for supported operating systems.

File system issues::
File system configurations that might cause conversion failures.

Multi-boot configuration::
The VM has multiple bootable operating systems. You must specify the correct root device path in the migration plan settings.

== Preparing for migration

Use Deep Inspection results to prepare VMs for migration:

Address critical concerns::
Resolve errors before you start the migration. Verify operating system support.

Configure root device for multi-boot VMs::
If Deep Inspection finds a multi-boot configuration, use the root device path in the inspection results. Configure the *Root device* setting in your migration plan.

Review file system compatibility::
Verify that all file systems are supported by {virt} and the target operating system version.

Plan resource allocation::
Use the disk and partition information to plan storage allocations in the destination environment.
