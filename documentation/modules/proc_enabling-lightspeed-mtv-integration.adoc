= Enabling Red Hat OpenShift Lightspeed integration for {project-short}

[role="_abstract"]
You can enable the Red Hat OpenShift Lightspeed integration with {project-short} to provide AI-assisted guidance for planning and troubleshooting VM migrations.

[NOTE]
====
Red Hat OpenShift Lightspeed integration with {project-short} is available as a Technology Preview feature. Technology Preview features are not supported with Red Hat production service level agreements (SLAs) and might not be functionally complete. Red Hat does not recommend using them in production.
====

.Prerequisites

* {ocp} 4.15 or later
* Cluster administrator access
* {project-short} 2.12 or later installed
* Red Hat OpenShift Lightspeed Operator installed and configured on your cluster
+
For information about installing and configuring Red Hat OpenShift Lightspeed, see the link:https://docs.redhat.com/en/documentation/red_hat_openshift_lightspeed[Red Hat OpenShift Lightspeed documentation].

.Procedure

. Deploy the {project-short} MCP server:
+
[source,terminal,subs="attributes+"]
----
$ oc apply -f https://raw.githubusercontent.com/kubev2v/kubectl-mtv/main/deploy/mcp-server.yaml
----
+
This creates the MCP server deployment and service in the `openshift-mtv` namespace. The MCP server provides the connection between Red Hat OpenShift Lightspeed and {project-short} resources.

. Register the {project-short} MCP server with Red Hat OpenShift Lightspeed:
+
// Reminder: Check with SME to verify endpoint is /mcp and not /sse
[source,terminal,subs="attributes+"]
----
$ oc patch olsconfig cluster --type merge -p '
spec:
  featureGates:
    - MCPServer
  mcpServers:
    - name: kubectl-mtv
      streamableHTTP:
        url: http://kubectl-mtv-mcp-server.openshift-mtv.svc:8080/mcp
        timeout: 30
        sseReadTimeout: 30
        enableSSE: true
        headers:
          Authorization: kubernetes
'
----
+
This patch enables the MCPServer feature gate and registers the {project-short} MCP server endpoint with Red Hat OpenShift Lightspeed.

.Verification

. Verify that the MCP server pod is running:
+
[source,terminal,subs="attributes+"]
----
$ oc get pods -n openshift-mtv -l app.kubernetes.io/name=kubectl-mtv-mcp-server
----
+
.Example output
[source,terminal,subs="attributes+"]
----
NAME                                      READY   STATUS    RESTARTS   AGE
kubectl-mtv-mcp-server-7d4b9c8f6b-x5z2p   1/1     Running   0          2m
----

. Verify that the OLSConfig includes the {project-short} MCP server:
+
[source,terminal,subs="attributes+"]
----
$ oc get olsconfig cluster -o jsonpath='{.spec.mcpServers[?(@.name=="kubectl-mtv")].name}'
----
+
.Example output
[source,terminal,subs="attributes+"]
----
kubectl-mtv
----

. Optional: Open the {ocp} web console and ask Red Hat OpenShift Lightspeed a question about {project-short}, such as "What are the prerequisites for warm migration?"

.Troubleshooting

* If the MCP server pod is not running, check the deployment logs:
+
[source,terminal,subs="attributes+"]
----
$ oc logs -n openshift-mtv -l app.kubernetes.io/name=kubectl-mtv-mcp-server
----

* If the OLSConfig patch fails, verify that the Red Hat OpenShift Lightspeed Operator is installed and the `cluster` OLSConfig resource exists:
+
[source,terminal,subs="attributes+"]
----
$ oc get olsconfig cluster
----
