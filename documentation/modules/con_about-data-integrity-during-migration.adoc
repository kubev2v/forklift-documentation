= Data integrity during migration

[role="_abstract"]
{project-first} detects and reports potential data issues as early as possible before migrated virtual machines (VMs) are handed off on {virt}.

Starting with {project-short} 2.12, {project-short} uses capabilities delivered in {rhel-first} 10.1 to strengthen data integrity and consistency during VM migration. These capabilities provide the following:

* File system consistency validation
* Reliable failure when migration operations encounter errors
* Internal diagnostics that help identify suspected storage problems

[id="what-mtv-validates-during-migration_{context}"]
== What {project-short} validates during migration

{project-short} strengthens migration in the following ways:

* Filesystem consistency at conversion boundaries
* Reliable failure when conversion or disk-copy operations encounter system or storage errors 
* Internal diagnostics during disk copy for support analysis

{project-short} also supports methods for minimizing migration cutover downtime while applying these integrity measures where the migration path supports them.
