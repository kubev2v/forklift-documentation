= About cold and warm migration

[role="_abstract"]
Cold migration is for virtual machines (VMs) that are shut down and works with {vmw} vSphere, {rhv-full}, {osp}, Open Virtual Appliances (OVAs) created by {vmw} vSphere, Microsoft Hyper-V, and remote {virt} clusters. Warm migration is for running VMs and works with {vmw} vSphere and {rhv-full}.

[NOTE]
====
OVA migration is validated for migrating supported guest operating systems exported from {vmw} vSphere. For third-party networking or security appliances, check with the vendor for native QCOW2 or Kernel-based Virtual Machine (KVM) images.
====
