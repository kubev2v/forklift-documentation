== Resolved issues 2.8.0

.MTV does not preserve out of range gateway IP address 
In earlier releases of {project-short}, the gateway IP address was not copied after migration when the gateway IP address of the network interface card (NIC) is out of the network interface subnet range. The issue affected migrations even if you enabled the `Preserve static IPs` setting. This issue has been resolved in MTV 2.8.0. link:https://issues.redhat.com/browse/MTV-2018[(MTV-2018)]
